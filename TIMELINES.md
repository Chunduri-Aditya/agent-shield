# Agent Shield — planning pointer

**Do not maintain a dated milestone calendar here.** Execution order lives only in
[`AGENT_SHIELD_TODO.md`](AGENT_SHIELD_TODO.md). Cross items off as they ship.

This file exists so older links and onboarding paths keep resolving. It carries
no schedule columns and no calendar checkpoints.

## Session discipline

Before ending a work session, update [`AGENT_SHIELD_TODO.md`](AGENT_SHIELD_TODO.md),
[`CONTEXT.md`](CONTEXT.md) for private notes when needed, and [`SESSION_STATE.md`](SESSION_STATE.md)
when parallel sessions need the latest handoff. Do not let checklist drift.

## Daily floor (every working day)

- One commit (even a stub)
- One job application
- One X or LinkedIn post
